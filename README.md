# IoT-Botnet：物联网僵尸网络
## 网络攻防与分布式拒绝服务研究套件
这是一套用于网络攻防技术研究的物联网僵尸网络教学演示程序，由“控制端”和“被控端”组成，主要用于在封闭的虚拟机环境中模拟僵尸网络（Botnet）的工作原理。编写这份代码的目的，不是为了制造破坏，而是为了揭开黑盒。只有理解了攻击者的武器是如何锻造的，我们才能更好地铸造盾牌。  如果你是学生，请利用这份代码学好 TCP/IP；如果你是运维人员，请利用它来测试你的防火墙规则；如果你是开发者，请参考它的并发模型但不要抄袭它的恶意逻辑。

适用对象: 网络安全研究员、高校信息安全专业、后端架构师、运维工程师

## 1. 项目概述与声明
### 1.1 项目简介
IoT-Botnet 是一套高度仿真、闭环隔离的僵尸网络（Botnet）教学模拟系统，由 **Go语言控制端(Controller)** 与 **C语言被控端Agent(Bot)** 双组件构成。整套系统仅允许在断网隔离虚拟机沙盒内运行，核心研究方向：
1. L3/L4/L7 全层级DDoS攻击底层协议实现原理；
2. TCP/UDP/TLS/QUIC/DNS 协议栈底层手工封包、校验和计算；
3. Linux 进程隐藏、守护进程、自启动持久化、权限对抗技术；
4. 僵尸网络C2通信、身份认证、断线自愈、DGA动态域名回连机制；
5. Go高并发服务、Goroutine异步调度、RBAC权限架构设计。

## 2. 系统整体架构、完整运行原理
### 2.1 系统架构拓扑
```
【管理员本地PC】
        ↓ SSH 22222（加密交互）
+---------------------------+
| 控制端 Controller (Go)    |
| 1. 内置SSH服务：用户登录、指令下发
| 2. TCP 9999 监听：接收所有Bot长连接
| 3. 连接池、心跳管理、权限校验
| 4. 异步广播队列、攻击任务调度
| 5. 实时统计、日志审计、用户热加载
+-----------TCP 9999双向长连接-----------+
                                          ↓
+----------------------------------------+
| 被控端 Agent (C语言 Linux)              |
| 1. C2通信模块：握手、心跳、断线重连、DGA |
| 2. 多线程攻击引擎：9种DDoS流量生成      |
| 3. 进程隐蔽、双进程守护、单例锁        |
| 4. 多系统自启动持久化植入               |
| 5. 日志记录、系统命令执行模块           |
+----------------------------------------+
        ↓ 攻击流量输出
【隔离靶机 / 模拟IoT路由器虚拟机】
```

### 2.2 完整全链路运行流程（含DGA故障自愈）
1. **控制端初始化启动**
    加载配置、用户文件、生成RSA密钥；开启9999（Bot接入）、22222（SSH管理）双端口；启动心跳、统计、用户重载、清理后台协程；创建PID文件防止多开。
2. **管理员SSH登录鉴权**
    通过22222端口连接内置SSH服务，校验账号、角色、单会话限制，进入交互式命令终端。
3. **Bot 初始化启动**
    Agent创建单例端口锁防止重复运行；执行双进程守护fork逻辑；重定向标准输出到/dev/null；读取内置C2静态IP常量，发起TCP连接。
4. **C2 三步身份握手认证（核心安全机制）**
    - Bot连接9999端口，发送字符串`hello`；
    - 控制端收到hello，返回内置全局`AUTH_KEY`密钥；
    - Bot校验服务端密钥与自身宏定义一致，发送本机hostname+版本号；
    - 校验通过，控制端下发唯一Agent ID，建立永久心跳长连接。
5. **常规心跳保活**
    Bot每30秒向控制端发送`PING`，服务端回复`PONG`；超过60秒无心跳直接清理失效连接。
6. **主C2断开 → DGA动态域名自愈流程（重点扩展说明）**
    - 当静态IP连接超时/被防火墙阻断，全局`global_stop`标记置为1，断开当前socket；
    - Agent调用`generate_backup_domain()`DGA函数，基于NTP北京时间+随机数哈希生成`.xyz`备用域名；
    - 循环尝试解析DGA域名、重新连接控制端9999端口；
    - 连续3次DGA域名连接失败，切回原始静态IP重试，循环交替直到恢复通信。
7. **管理员下发攻击指令**
    SSH终端输入`attack`命令，控制端校验用户角色、速率/线程/时长限流参数；合法指令推入异步广播channel，分发给所有在线Bot。
8. Bot解析指令启动多线程攻击
    读取攻击模式、目标、端口、包大小、速率参数，批量创建pthread发包线程，按照指定pps速率持续发送构造数据包。
9. 任务终止流程
    管理员执行`stop`，控制端广播停止指令；Bot收到后置全局停止标记，销毁所有攻击线程，释放raw socket资源。
10. 持久化驻留（可选自动执行）
    Agent首次运行时自动检测root权限，自动写入systemd/openwrt/crontab开机脚本，设备重启自动重新执行、重新连接C2。

## 3. 控制端 Controller 完整技术与功能详解
### 3.1 基础技术栈
- 开发语言：Go 1.18+
- 并发核心：Goroutine + Channel 异步队列 + sync.Map 并发安全连接池
- 通信协议：
  1. 私有明文TCP（9999，Bot C2通信，内置密钥鉴权）
  2. 自研内置SSH服务（2222，RSA加密管理员交互，不依赖系统sshd）
- 第三方依赖：zap结构化日志、golang.org/x/ssh、color终端彩色输出
- 系统能力：PID锁、信号捕获、优雅关闭、IP黑白名单、定时后台任务

### 3.2 全部核心模块细分功能
#### 模块1：内置独立SSH管理服务
1. 程序启动时自动生成2048位RSA主机密钥，独立SSH监听端口；
2. 账号密码鉴权，区分Admin/User/Guest三级角色；
3. 单账号单会话限制：同一账号禁止多地同时登录；
4. 交互式PTY终端，支持退格、窗口大小自适应、彩色命令输出；
5. 会话超时自动注销（30分钟无操作断开）；
6. 支持logout/exit安全释放会话资源。

#### 模块2：RBAC 三级权限控制系统
角色划分与硬编码限制：
1. Guest游客：无任何操作权限，仅能登录查看提示；
2. User普通用户：仅可执行help/stats/attack/logout；参数强限制：
   - 最大单任务线程：1
   - 每秒发包上限：10000 PPS
   - 单攻击最长60秒，攻击结束强制120秒冷却；
3. Admin管理员：全功能开放，无速率/时长限制，可执行stop、shell远程系统命令。
附加机制：单任务互斥，全局同一时间仅能运行一条攻击任务。

#### 模块3 Bot节点连接管理池
1. sync.Map存储全部在线Agent，支持百万级并发读写；
2. 每条连接独立读写协程，互不阻塞；
3. 连接状态标记：已连接/测试中/断开/异常；
4. 定时清理心跳超时离线节点，自动回收资源；
5. 支持IP黑白名单，拦截指定IP接入；
6. 记录每个Agent唯一ID、主机名、程序版本、上线时间。

#### 模块4 异步指令广播调度系统
1. 无阻塞Channel消息队列，缓冲10000条指令，海量节点下发不卡顿；
2. 支持全局广播所有Bot，也支持指定单个Agent下发指令；
3. 指令下发异步协程分发，不阻塞SSH交互终端；
4. 任务持久记录，存储任务ID、创建人、起止时间、攻击参数。

#### 模块5 用户配置热加载机制
1. 每10秒自动读取`user.txt`用户配置文件；
2. 新增账号、修改密码、调整角色无需重启控制端；
3. 检测已删除用户，在线账号标记禁用、会话超时断开；
4. 读写分离锁保证多协程读取用户信息无竞态。

#### 模块6 实时监控与日志审计
1. 定时5秒输出全局统计：运行时长、总连接数、在线Bot、测试中节点、总指令数；
2. 结构化JSON持久日志，存储攻击任务、登录、认证、节点上下线事件；
3. PID文件锁防止重复启动；
4. 捕获SIGINT/SIGTERM信号，执行优雅关闭：停止攻击、断开所有连接、清理PID、同步日志。

## 4. 被控端 Agent (C语言) 完整功能、运行原理
### 4.1 基础技术栈
- 标准C99，纯Linux POSIX接口，无第三方动态依赖；
- 核心头文件：pthread多线程、raw socket原始套接字、netinet协议头、prctl进程控制；
- 编译模式：支持静态编译，无依赖二进制，可直接在嵌入式路由器运行；
- 运行硬性前置：必须ROOT权限（原始套接字、修改系统服务、绑定低端口）。

### 4.2 核心模块1：C2通信 + DGA动态域名自愈（重点详解）
#### 4.2.1 C2基础通信流程
1. 程序启动读取宏定义静态C2 IP `CTRL_SERVER_IP`，发起TCP连接；
2. 执行三步身份认证hello-key握手，校验服务端AUTH_KEY一致性；
3. 主线程持续阻塞读取TCP流，区分PING心跳、start攻击、stop停止、shell命令；
4. 收到停止标记立刻销毁全部攻击线程，释放套接字。

#### 4.2.2 DGA域名生成完整运行逻辑（代码逐行拆解）
DGA全称：Domain Generation Algorithm 动态域名生成算法，作用：静态C2 IP被防火墙封禁/服务器下线时，自动生成备用域名回连，实现断线自愈。
完整执行步骤：
1. **时间源校准**
    函数`get_ntp()`轮询阿里云、Cloudflare、公网NTP服务器获取北京时间UTC+8；
    NTP读取失败则使用本地系统时间兜底，保证哈希种子统一。
2. **种子拼接**
    格式化时间字符串`%Y%m%d%H%M%S`（年月日时分秒），拼接固定字符串`_hellobot_` + 随机uint整数，作为哈希输入原文。
3. **DJB2哈希计算**
    使用轻量无依赖DJB2哈希函数：`hash = hash * 33 + 字符ASCII码`，遍历全部字符串生成32位无符号哈希值。
4. **哈希转8位十六进制字符串**
    将哈希数值格式化输出为8位小写16进制字符，作为域名主体。
5. **拼接固定后缀**
    硬编码后缀`.xyz`，拼接得到完整备用域名，例如`8f2d59c1.xyz`。
6. **连接重试逻辑**
    - 静态IP连续连接失败 → 调用DGA生成域名；
    - 解析域名、发起TCP连接9999；
    DGA域名连续3次失败，切换回原始静态IP循环重试；
    - 每60秒刷新一次DGA种子（时间变化，域名同步更新）。
7. 日志记录生成的域名，便于实验中抓包识别DGA流量。

DGA核心代码逻辑对应函数：`generate_backup_domain()`、`simple_hash()`、`get_ntp()`。

### 4.3 核心模块2：9类全层级DDoS攻击引擎
统一指令格式：
`start <mode> <目标> <端口> <pps> <持续秒> <线程数> <包大小> [扩展参数]`
所有攻击均使用pthread创建独立发包线程，线程内部循环封包、计算校验和、sendto发送。
1. **MODE_SYN L4 SYN泛洪**
    手工构造完整IPv4头部 + TCP SYN头部，自定义TTL、IP标识；
    固定本机出口真实IP（通过临时socket获取本机网卡IP），随机源端口；
    填充自定义长度随机载荷，计算IP/TCP双层校验和；
    利用raw socket直接下发，不经过操作系统TCP协议栈，打满目标半连接队列。
2. **MODE_ACK L4 ACK泛洪**
    手工构造纯ACK报文，无SYN标记，随机seq/ack序列号；
    大量空ACK包迫使防火墙、服务器持续查表消耗CPU。
3. **MODE_TCP 普通TCP流量**
    多并发TCP长连接，完整三次握手后持续发送随机二进制载荷；
    模拟正常业务流量，难以基于特征过滤。
4. **MODE_TCP_CONN TCP长连接耗尽**
    维持上千条永久不关闭TCP连接，持续循环发送数据；
    占用目标文件描述符、内存连接表，直接导致服务无法新建连接。
5. **MODE_UDP UDP带宽泛洪**
    手工构造UDP头部+随机载荷，大尺寸数据包打满链路带宽，常用于DNS服务器轰炸。
6. **MODE_HTTP L7 CC攻击**
    复用TCP持久连接，构造标准完整HTTP1.1 GET请求；
    支持自定义Host头、URL路径、多并发连接；
    可自定义请求包大小，填充大量扩展HTTP头消耗Web服务算力。
7. **MODE_TLS TLS握手泛洪**
    构造标准ClientHello握手报文，随机填充扩展字段；
    高频发起握手，迫使服务器频繁RSA/ECC加解密，消耗CPU。
8. **MODE_DNS DNS查询攻击**
    构造标准DNS A记录查询包，随机查询域名；
    持续轰炸DNS服务器，耗尽解析并发上限。
9. **MODE_QUIC HTTP/3 UDP泛洪**
    基于UDP封装QUIC初始握手包，随机连接ID；
    绕过传统TCP防火墙，新一代L7 DDoS模拟。
10. **MODE_SHELL 系统命令执行**
    不产生流量，调用system()执行下发的Linux命令，用于内网信息采集。

### 4.4 核心模块3 进程隐蔽与对抗全套功能
1. **进程名伪装**
    使用`prctl(PR_SET_NAME, "sshd")`修改内核进程名；
    覆盖argv[0]内存，ps/aux/top查看均显示sshd后台服务。
2. **双进程守护架构**
    程序启动两次fork：
    第一层fork脱离终端成为后台父监控进程；
    第二层fork生成工作子进程；
    父进程循环waitpid，子进程崩溃、被kill、异常退出自动重启Agent，永久保活。
3. **单例互斥锁（端口锁）**
    绑定本地回环127.0.0.1:60005端口，仅允许单实例运行；
    重复启动直接退出，避免多进程暴露特征。
4. 标准输出净化
    守护进程模式将stdin/stdout/stderr全部重定向至`/dev/null`，无控制台输出残留。
5. PID文件文件锁
    `/var/run/stress_agent.pid`写入当前PID，flock文件互斥，防止多开冲突。
6. 本地日志持久
    全部运行日志写入`/var/log/stress_agent.log`，记录连接、攻击启动/停止、DGA域名、错误信息。

### 4.5 核心模块4 跨平台自动持久化驻留
Agent检测root权限后自动复制自身二进制到系统目录，并写入开机自启脚本，支持三类Linux系统：
1. Systemd系统（Ubuntu/CentOS）
    生成`/etc/systemd/system/stress_agent.service`服务单元，设置Restart=always崩溃自动重启；
    执行daemon-reload、enable开机启动。
2. OpenWrt路由器系统
    生成procd开机脚本`/etc/init.d/stress_agent`，设置respawn自动重启。
3. 通用Linux（无systemd）
    方案1：追加rc.local开机执行；
    方案2：添加crontab @reboot重启任务；
安装完成后设备重启自动运行Agent，自动连接C2。

### 4.6 核心模块5 安全校验与限制
1. C2双向密钥认证，无匹配KEY直接断开连接；
2. 发包线程严格按照pps微秒延时控速，避免瞬间打满本机带宽；
3. 包大小强制限制：最小64字节，最大65535字节，超出自动截断；
4. 攻击持续时间倒计时，到达时长自动停止所有线程；
5. 全局`global_stop`原子标记，多线程安全快速终止流量。
